## 🔒 Security Best Practices

### API Key Management
✅ DO:
- Store in `.env` file locally
- Use environment variables in Docker
- Different keys for dev/prod
- Rotate keys monthly
- Monitor API console for unusual activity

❌ DON'T:
- Hardcode in source code
- Commit `.env` to Git
- Share via email/chat
- Use same key for all environments
- Ignore rate limit warnings

### Git Security
```bash
# Ensure .gitignore has:
.env
.env.dev
.env.prod
node_modules/
__pycache__/

# Remove if previously committed
git rm --cached .env
git commit -m "Remove .env from tracking"
```

## 🆘 Troubleshooting

### Issue: Backend won't start
```bash
# Check logs
docker-compose logs backend

# Common fixes:
# 1. Missing API_KEY - ensure .env file exists
# 2. Port 5000 in use - change in docker-compose
# 3. Missing dependencies - rebuild: docker-compose build --no-cache
```

### Issue: "GEMINI_API_KEY not set"
```bash
# Verify .env file
ls -la .env
grep GEMINI_API_KEY .env

# Restart backend
docker-compose restart backend
```

### Issue: Invalid API key error
```bash
# Verify at: https://makersuite.google.com/app/apikey
# Check Generative Language API is enabled
# Copy exact key without spaces

# Update .env and test
docker-compose restart backend
curl http://localhost:5000/health
```

### Issue: No MCQs generated
```bash
# Check backend logs
docker-compose logs backend | grep -i gemini

# Verify API key works by testing directly
python << 'EOF'
import google.generativeai as genai
genai.configure(api_key="YOUR_KEY")
model = genai.GenerativeModel("gemini-pro")
response = model.generate_content("Write a test question")
print(response.text)
EOF
```
